## 「プットが急増＝暴落の前兆」は本当か

暗号資産の相場を見ていると、「プットの買いが急増した」「恐怖指数が跳ねた」という投稿を目にします。そして多くの場合、その後に「暴落が近い」という話がセットで語られます。オプション市場は、参加者がお金を賭けて将来に備える場所なので、そこに現れる恐怖には情報がありそうに思えます。

ただ、その直感は本当でしょうか。恐怖が高まっていることと、実際に暴落が来ることは、どれくらい結びついているのか。この記事では、次の問いをデータで確かめます。オプション市場の恐怖指標は、その後の下落について、どこまで当てにできる情報を持っているのか。

先に結論の方向だけ言うと、恐怖指標は「暴落の予言者」ではなく「リスクの温度計」でした。恐怖が高いときは確かにその後の下落が深くなりやすいのですが、暴落そのものを精度よく当てられはしません。順番に見ていきます。

## 市場の恐怖を測る3つの指標

オプション市場から「恐怖」を読むとき、よく使われる指標が3つあります。

- プット・コール比率: プットの取引量（または建玉）を、コールの取引量（または建玉）で割った値。プットの取引が厚いほど、下落への備えが多いと解釈されがちです
- IVスキュー: 権利行使価格の離れたプットとコールで、予想変動率（IV）を比べた差。下落側のプットのほうが割高なら、市場は下落をより警戒しています
- 建玉（OI）: いま市場に残っている契約の量。取引量が「その日の勢い」なら、建玉は「積み上がったポジション」です

ここで一つ注意があります。建玉は、買い手と売り手が必ず同数いる契約の数です。だからプット・コール比率が高くても、それだけで「参加者が弱気」とは読めません。誰かがプットを買っているなら、必ず誰かが売っています。比率は雰囲気の目安であって、方向を直接示す数字ではない、と押さえておいてください。

そのうえで、いまのビットコインのオプション市場（2026年7月22日時点のDeribit）を集計すると、こうなっていました。プット・コール比率は建玉ベースで0.44、取引量ベースで0.25です。数の上ではコールのほうが多く、少なくとも「プットだらけ」の状態ではありません。

予想変動率を権利行使価格ごとに並べると、下落側の傾きが見えます。

![現在のIVスマイルとスキュー](/images/btc-options-fear-predict-crash/fig1_smile.png)

横軸が権利行使価格、縦軸が予想変動率です。中心（原資産、いまはBTCの現物価格）から離れるほどIVが高くなる形はスマイルと呼ばれます。そして左側、価格が下がる方向のプット（赤）のほうが、右側の上昇方向のコール（青）より高くなっています。ためしに原資産から15%ほど下の行使価格のプットと、15%ほど上のコールのIVを比べると、下のほうが約6.7ポイント高い値でした。これがスキューで、市場が下落方向の備えにより高い値段を払っていることを表します。なお、これはデルタを厳密に揃えた比較ではなく、行使価格でざっくり取り出したおおよその値です。

ここまでは「いまの市場が下落をやや警戒している」という状態の話です。では、この恐怖は、その後の暴落を当てられるのでしょうか。

## なぜ「プットが多い＝暴落」とは限らないのか

検証に入る前に、指標をそのまま信号にできない理由を押さえておきます。プットが買われていても、それが下落予想とは限らないからです。

- 現物のヘッジ: ビットコインを現物で持っている人が、下落に備えてプットを買う。これは弱気予想ではなく保険です
- 組成取引の一部: コラーのように、現物とプットとコールを組み合わせた取引の一部として、プットが売買される
- 満期や権利行使価格の偏り: 特定の満期に建玉が集中しているだけのことがある
- 取引所ごとの違い: 参加者の顔ぶれや流動性が違えば、同じ比率でも意味が変わる

だから比率やスキューを単独で「暴落シグナル」として読むのは危うい、というのが出発点です。

そのうえで、本来なら「過去にプットが増えた局面やスキューが立った局面のあと、実際に暴落したか」を長期間で検証したいところです。ところが、ここで現実的な壁がありました。過去のスキューやプット・コール比率の時系列を、長期にわたって公開APIから取ることができません。Deribitの時刻指定つき取引履歴は、公式仕様では直近24時間ぶんしか返らず（実際に叩いても、数日前より古い時刻はすべて空でした）、何年も前の権利行使価格ごとのIVを再現するのは、公開データだけでは難しいのです。

そこで本稿では、5年ぶんの履歴が取れる指標を代わりに使います。前回も登場したDVOLです。DVOLはオプション市場が見積もる「今後30日の予想変動率」を1本の数字にした指数で、市場全体の身構え、いわば恐怖の水準として読めます。方向性までは含みませんが、「市場がどれくらい大きな変動を覚悟しているか」の温度計としては十分です。以降は、このDVOLを恐怖の代理として、暴落を当てられるかを見ていきます。

## 恐怖が高い局面のあと、下落は深いのか

まず素直な問いから始めます。DVOLが高い（＝市場が身構えている）局面のあとは、その後の下落が深いのでしょうか。

検証の作り方はシンプルです。各日について、その日の終値から、その後30日でいちばん安い終値までどれだけ下げたか（ここではこれを下落率と呼びます）を計算し、その日のDVOLの高さ別に比べます。前回学んだとおり、ここは「その日の翌日から前を向いて30日」を正しく取るのがポイントです。

```python
import pandas as pd

# 各日 t について「その翌日から30暦日先まで」の、最安の終値への下落率（%）
def forward_drawdown(t):
    seg = df.loc[(df["date"] > t) & (df["date"] <= t + pd.Timedelta(days=30)), "close"]
    p0 = df.loc[df["date"] == t, "close"].iloc[0]
    return (seg.min() / p0 - 1) * 100 if len(seg) >= 20 else None  # 窓が満たない末尾はNaNで除外

df["fwd_dd30"] = df["date"].map(forward_drawdown)
df["crash30"] = df["fwd_dd30"] <= -20          # 30日で20%超下げたら「暴落」とする（NaNは集計から除く）
```

厳密なピークからの最大下落（マックスドローダウン）ではなく、その日を起点にどこまで沈んだかを終値ベースで見る、という単純な作りです。ここでは、その後30日で20%を超えて下げた場面を「暴落」と呼ぶことにします。検証に使えたのは2021年3月から2026年7月の1926日ぶんで、暴落が起きた割合は全体の12.9%でした。8日に1日くらいは、その先に大きめの下げが控えていた計算です。

これをDVOLの高さで3つの局面に分けて比べると、はっきり差が出ます。

![DVOLと暴落が起きた局面](/images/btc-options-fear-predict-crash/fig2_dvol_crash.png)

赤い点が「この後30日で20%超下げる」起点です。DVOLが高く跳ねている時期に、赤が固まっているのが見えます。局面別の平均ドローダウンにすると、こうなります。

![DVOL局面別の平均ドローダウン](/images/btc-options-fear-predict-crash/fig3_bucket_dd.png)

低DVOLと中DVOLの局面では、その後30日の平均下落率はマイナス7%台でした。ところが高DVOLの局面ではマイナス13.0%と、下落が明らかに深くなっています。20%超の下げが起きた割合も並べてみます。

![DVOL局面別の暴落率](/images/btc-options-fear-predict-crash/fig4_crashrate.png)

低DVOLで10.0%、中DVOLで8.6%だった暴落率が、高DVOLでは20.2%へ跳ね上がります。低や中の局面と比べると、高恐怖の局面では暴落の頻度がおよそ2倍です。ここまでを見る限り、恐怖が高い局面はその後の下落が深く、暴落も多い、というのは本当でした。恐怖は、リスクの高まりを映しています。

## では、暴落を「当てられる」のか

ただ、「下落が深くなりやすい」ことと「暴落を当てられる」ことは、同じではありません。高DVOLの局面でも、暴落しなかった日のほうが8割を占めているからです。予報としてどれくらい役に立つのかを測るには、当たりと外れの両方を見る必要があります。

ここで使うのがROCという見方です。

:::message
ROCは、ある指標をどの高さで「警報」に切り替えても、検出率（暴落をちゃんと当てた割合）と誤検知率（暴落でないのに警報を出した割合）がどう動くかを描いた曲線です。曲線の下の面積をAUCと呼び、1に近いほど当たる指標、0.5ならコイン投げと同じ、を意味します。
:::

DVOLの水準で「この後30日の暴落」を当てにいくと、こうなりました。

![恐怖指標のROC曲線](/images/btc-options-fear-predict-crash/fig5_roc.png)

青い線がDVOLの水準です。でたらめを表す点線（AUC0.50）より上にはありますが、AUCは0.60でした。コイン投げよりはマシですが、暴落を当てる指標と呼べるほどではありません。実感としては、警報のしきい値を上げれば見逃しが増え、下げれば「暴落でないのに警報」が増える、というトレードオフから抜け出せません。恐怖が高いとリスクは上がるものの、それを「暴落が来る」という予報に変えようとすると、精度はかなり心もとないのです。なお、重複した窓を数えているぶんデータは互いに似ているので、この0.60という値も幅を持って読むべきものです。

## 恐怖の急上昇は、暴落の前兆になるのか

もう一歩踏み込みます。「恐怖の水準」ではなく「恐怖の急上昇」なら、暴落の前兆になるのではないか。プットが急に買われた、恐怖指数が急騰した、という話はまさにこれです。

そこでDVOLの7日間の変化を信号にして、同じように暴落を当てにいきました。結果は、先ほどのROC図の緑の線です。AUCは0.45で、でたらめの0.50を下回りました。少なくとも「恐怖の急上昇が大きいほど、この後に暴落する」という向きには、当たっていないということです。

なぜこうなるのか。DVOLが急上昇するのは、暴落の前ではなく、暴落の最中や、下げ切った底の近くであることが多いから、と考えるとつじつまが合います。価格が崩れてから市場が慌てて身構えるので、急上昇を見てから動いても遅い、というわけです。ただし今回は急上昇の起点と価格の道のりを一つずつ追ったわけではないので、これはAUCの結果と整合する解釈にとどめておきます。

高DVOLの局面のその後を見ると、方向が一つに決まらないことがはっきりします。

![高DVOL局面のその後30日リターン](/images/btc-options-fear-predict-crash/fig6_high_dvol_outcome.png)

高DVOLの局面から30日後のリターンを分布にしたものです。平均はマイナス2.2%とわずかに沈んでいますが、山は0の両側に広く散らばっています。実際、高DVOLの局面のうち約40%は、30日後にはむしろ価格が上がっていました。恐怖が高いからといって、その先が下とは限らない。同じ「恐怖が高い」局面でも、その後は上と下のどちらにもわかれる、というのがデータの姿でした。

だから恐怖指標は、方向を教えてくれる予言者ではありません。これから大きく動きやすい、という不確実性の高まりを映す、温度計のような存在です。

## わかったことと、この検証の限界

この5年のビットコインで分かったことを整理します。

- 恐怖（DVOL）が高い局面は、その後の下落が深く（平均マイナス13%、平時はマイナス7%台）、暴落の頻度も低や中の局面のおよそ2倍でした。リスクの高まりは映しています
- しかし暴落を当てる精度は弱く（AUC0.60）、リスクは上がっても、予報にはなりにくいです
- 恐怖の急上昇は、その向きでは当たりませんでした（AUC0.45）。急上昇は暴落の前でなく、最中や底で起きると考えると整合します
- 高恐怖の局面は、その後に下がることもあれば、上がることもあります（40%は30日後に上昇）
- 恐怖指標は予言者ではなくリスクの温度計です。単独の売買シグナルにすると外れます

「プットが急増したから暴落する」というより、市場が大きな変動を覚悟している、くらいに読むのが、データに合った受け止め方でした。

最後に、この検証の限界も書いておきます。

- 今回「恐怖」に使ったのはDVOLで、これは上下両方向を含む変動率の水準です。下落方向に絞った恐怖（IVスキューやプット・コール比率）の長期時系列は公開APIから取れなかったため、今回は現在のスナップショットを教材として示すにとどめています。方向性の恐怖に予知力があるかは、別途データを貯めて確かめる必要があります
- DVOLは上下を区別しない指標なので、今回見たのは「下落の深さとの関係」だけです。上昇方向の大きな動きや、実現した変動そのものとの関係は、この記事では検証していません
- 30日の窓を毎日ずらして重ねているため、隣り合う日のデータは似ています（自己相関、つまり近い日どうしの値が連動すること）。AUCや割合を、統計的に確立した性質のように強くは読みません
- これはあくまで、ある取引所の、ある期間の、ある定義での結果です。暴落を「30日で20%超の下げ」と決めたのも一つの選び方で、基準を変えれば数字は動きます

恐怖指標は、暴落を当てる道具としては頼りない一方で、リスクがどれくらい高まっているかを測る目安にはなります。予言を期待すると裏切られる、という距離感で付き合うのがよさそうだ、というのが今回の結論でした。

次回は、この「恐怖に値段が付く」市場で、保険を売る側にまわる戦略（毎月プレミアムを受け取るカバードコールなど）が、現物を持つだけと比べて本当に得なのかを検証します。
