## 自律エージェントは「どこで動かすか」でつまずく

OpenClaw や Hermes Agent のような自律型AIエージェントは、常駐させて初めて価値が出る。24時間動いて、定期実行して、Slack や Telegram から指示を受ける。この使い方をしようとすると、最初にぶつかるのが「どこで動かすか」だ。

選択肢は2つある。手元のPCを常時起動しておくか、VPSを借りて `curl` でセットアップするか。前者は電気代とファンの音、寝ている間もPCを開けっぱなしにする気持ち悪さがある。後者はサーバーの知識が要るし、SSHでログインしてインストール、環境変数、プロセス管理と、動かすまでの手数が地味に多い。Hermes の自己ホストの手順は別記事に書いたが、正直、誰にでも勧められる作業ではない。

このあたりで止まっている人は多いと思う。自分もそうだった。そこで試したのが、ロリポップ（GMOペパボ）の[AIエージェントクラウド](https://lolipop.jp/ai/agent-cloud/)だ。結論から言うと、常駐先の悩みがそのまま消えた。

## AIエージェントクラウドとは

自律AIエージェントをクラウド上で動かすためのサービスで、レンタルサーバーで知られるロリポップが2026年に始めた。最大の特徴は、エージェントの構築から運用までを**すべてブラウザのGUIで完結できる**点にある。コマンドもプログラミングも要らない。

対応エージェントは3種類ある。

| エージェント | 位置づけ | 作成数 |
| --- | --- | --- |
| OpenClaw | 人格・スキル・連携先を自由に組む汎用型 | 複数可 |
| Hermes Agent | 記憶と実行を繰り返して最適化する自己学習型 | 1契約1体 |
| NanoClaw | 機密情報をAIから隔離するセキュリティ特化型（2026年7月対応開始） | 1契約1体 |

OpenClaw と Hermes の性格の違いは、そのまま公式の位置づけに表れている。汎用的に何でも組みたいなら OpenClaw、使うほど自分に馴染ませたいなら Hermes、という住み分けだ。この2つを、後述するようにフォームとチャットだけで立ち上げられる。

## 良かった点1：OpenClaw も Hermes も、GUIの設定だけで動く

一番効いたのはここだ。

OpenClaw を自前で立てようとすると、ランタイムとゲートウェイとスキルの構成を理解して、サーバー上で組み上げる必要がある。Hermes Agent は `curl` 一本で入るとはいえ、VPSの契約とLLMのAPIキー設定、常駐の管理は自分の仕事になる。どちらも「動かせる人には簡単、そうでない人には壁」という類のものだ。

AIエージェントクラウドは、この構築部分をブラウザの画面に置き換えている。エージェントの種類を選び、あとはチャットで指示するだけで設定が進む。人格やスキル、連携先を決めるのに、JSONを書く必要はない。エージェントの人格は `SOUL.md` という画面から編集する形になっていて、コードというより「設定シート」を書く感覚に近い。

![エージェントの人格を定義するSOUL.mdの編集画面](https://static.zenn.studio/user-upload/55d1834acbf1-20260713.png)

とはいえ、細かく詰めたくなったときの逃げ道もある。詳細設定では `config.yaml` を直接編集でき、使用モデルやプロバイダを自分で指定できる。GUIで始めて、慣れたらここで詰める、という進み方ができる。

![config.yamlを直接編集できる詳細設定画面](https://static.zenn.studio/user-upload/638aad0c8a36-20260713.png)

自分でVPSにHermesを入れた経験があると、この差はよくわかる。やっていること自体は同じでも、SSHとコマンドが画面のクリックとチャットに変わるだけで、始めるまでのハードルがまるで違う。公式は「3分でAIエージェントが起動」と書いているが、触った感覚でもそれに近かった。

ここは言い換えると、**これまでコマンド操作で諦めていた人でも自律エージェントに手が届く**ということだ。エンジニアでなくても、Slack を使っている中小チームが「定例のリマインドと議事録整理をエージェントに任せる」くらいのことは、この画面だけで組める。

## 良かった点2：月1,200円は「サーバー代」として妥当

料金は月額1,200円の単一プランで、全機能が使い放題になる。契約日から1ヶ月ごとの更新で、使った分だけ課金が膨らむ従量制ではない。この「定額で青天井にならない」設計は、個人が試すうえで安心材料になる。

この数字をどう見るか。自律エージェントを常駐させるとは、要するに24時間動くサーバーを1台持つことだ。その視点で1,200円を眺めると、むしろ安い部類に入る。サーバー・脆弱性対策・自動更新・ユーザーごとに独立したサンドボックス環境まで込みでこの価格になる。

専用PCを用意する案とも比べてみた。3つの選び方を並べるとこうなる。

| 方法 | 初期コスト | 月コストの目安 | 手間 |
| --- | --- | --- | --- |
| 手元PCを常時起動 | 手持ちなら0円 | 電気代 数百円〜 | 電源つけっぱなし・故障リスク |
| VPSに自前構築 | 0円 | 1,000円前後＋LLM代 | SSH・セットアップ・保守は自分 |
| AIエージェントクラウド | 0円 | 1,200円＋LLM代 | 構築・保守はサービス側 |

中古でエージェント用のマシンを買って24時間つけっぱなしにすると、本体代に加えて電気代が乗り続け、故障と置き場所の問題も抱える。月1,200円で「電源を落とさなくていいサーバーと保守」がまとめて手に入ると考えると、専用PCより素直な選択肢だと思う。

しかもテスト用のLLM無料枠が付いていて、APIキーなしでお試しできる。エージェントを触ったことがない段階で、自分の OpenAI や Anthropic のキーを用意しなくても動きを確かめられるのは、最初の一歩として親切だ。本格運用に移るときに高性能モデルへ切り替えればいい。

## 既存のチャットツールにそのまま組み込める

もう一つ地味に効くのが、Slack・Discord・Telegram との連携が管理画面から設定できる点だ。

自律エージェントの価値は、結局「どこで受け取れるか」で決まる。ターミナルの前に座っていないと使えないエージェントは、日常には馴染まない。普段チームで使っている Slack にエージェントを住まわせて、そこに話しかければ動く。この形になって初めて、業務のなかで自然に使うようになる。

連携は管理画面の「チャネル連携」から、Discord・Slack・Telegram をそれぞれ接続するだけだ。自分は Hermes Agent を Discord につないで使っている。

![チャネル連携画面。Discordが接続済みになっている](https://static.zenn.studio/user-upload/67e7e09c4ab3-20260713.png)

チャンネルで話しかければ動く。さらに、管理画面には「定期実行」のメニューがあり、時間を決めてタスクを自動で走らせる設定もGUIでできる。cronを書く必要はない。自分は「毎朝ニュースを要約してDiscordに投げる」を定期実行に登録していて、放っておくだけで毎朝まとめが届く。外出先から Telegram で「あの資料まとめといて」と単発で投げることもできる。既にチャットツールを使っているなら、新しいアプリを覚える必要がないのは大きい。

![時間を決めてタスクを自動実行できる定期実行の設定画面](https://static.zenn.studio/user-upload/0ebbad137974-20260713.png)

## 他のノーコードAI系とは、そもそも狙いが違う

「ノーコードでAI」と聞くと Dify や Copilot Studio、Make あたりを思い浮かべる人もいるはずだ。ただ、これらとロリポップは狙っている場所が違う。

Dify や Copilot Studio は、チャットボットやワークフローを**作る**ためのビルダーだ（[ノーコードAIエージェント比較の整理](https://keiei-digital.com/column/ai-agent/free-tools/)）。対してロリポップのAIエージェントクラウドは、OpenClaw や Hermes という**既にある自律エージェントを、常駐させて動かす**ための場所を提供する。前者が「アプリを組む」なら、後者は「エージェントに住所を与える」。

料金の考え方も違う。多くのノーコード系はユーザー単位や従量課金で、使い込むほど費用が読みにくくなる。ロリポップは月1,200円の定額で全機能という割り切りなので、コストを最小に固定してAI自動化を始めたい人には向く。「まず一体、低コストで常駐させて感触を掴む」という入り方に、価格設計が素直に噛み合っている。

## どんな人に向くか

正直に言うと、サーバーの構築と運用が苦にならない人には、このサービスは要らない。VPSにHermesを入れて自分で管理したほうが自由度は高いし、コストも詰められる。

向いているのは、次のような人だ。ITの専門知識はないが業務を自動化してみたい個人や中小チーム、手元のPCを24時間つけっぱなしにしたくない人、OpenClaw や Hermes をまず動かして感触を掴みたい人、そしてサーバーの保守を自分でやりたくない人。要するに「興味はあるのに、動かす場所で止まっていた層」だ。

自分は「Hermesを試したいが、常駐管理まではやりたくない」という温度感だったので、ちょうど間を埋めてくれた。

## 使う前に知っておくといいこと

過度に持ち上げても仕方がないので、フラットに書いておく。

サーバーのスペック（CPU・メモリ）は公開されていない。サーバー側で管理する設計なので、重い処理をどこまで回せるかは実際に使って確かめる範囲になる。セキュリティ特化型の NanoClaw は2026年7月対応開始で比較的新しく、枯れた安定を最優先するなら少し様子を見る判断もある。そして自由度は自前VPSに一歩譲る。GUIで完結する代わりに、コマンドで何でも触りたい人には物足りない場面もありうる。

こうした割り切りを理解した上でなら、「常駐先の悩みを月1,200円で外注する」という選び方は十分に合理的だ。

## まとめ

- 自律AIエージェントは常駐先で最初につまずく。手元PCの常時起動もVPSのセットアップも、それぞれ面倒がある
- ロリポップのAIエージェントクラウドは、OpenClaw と Hermes をブラウザのGUIとチャット設定だけで常駐させられる。コマンドもJSONも要らない
- 月1,200円の定額・全機能で、専用PCやVPSと比べても素直な選択肢。テスト用LLM無料枠でAPIキーなしから試せる
- Slack・Discord・Telegram に組み込め、管理画面から定期実行も設定できる。既存のチャットツールの中で、放っておいても回る運用が組める
- 他のノーコードAI系が「アプリを作る」道具なのに対し、これは「自律エージェントに住所を与える」サービス

自律エージェントを「試したいのに常駐先で止まっている」なら、月1,200円でOpenClawかHermesを一体立ち上げて、DiscordやSlackにつないでみるのがいい。念のため補足すると、無料なのはテスト用のLLM枠（＝自分のAPIキーなしで動きを試せる）だけで、サーバー自体は月1,200円だ。ここは勘違いしないようにしたい。合うかどうかは、動かしてみれば数分で分かる。
