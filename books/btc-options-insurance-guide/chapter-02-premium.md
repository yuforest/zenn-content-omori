## 「高い」というのは、何と比べて高いのか

序章で、オプションは値動きの保険のようなものだ、と話しました。この章からは、その保険料が本当に妥当だったのかを、データで確かめていきます。

よく言われるのが「ビットコインのオプションは高い」という話です。値動きが激しいぶん、保険料にあたる金額（プレミアム）も大きくなります。ただ、ここで一つ疑問がわきます。高いというのは、何と比べて高いのでしょうか。プレミアムが大きくても、実際にそれ以上の値動きが起きるなら、その保険料は妥当だったことになります。

そこでこの章では、次の問いを実データで検証します。**市場が見積もった「今後30日の値動き」は、その後に実際に起きた値動きより高かったのか**。言い換えると、ビットコインの保険料は、平均的に払いすぎだったのか、という問いです。

検証にはDeribit（暗号資産オプションの主要取引所）の公開データを使い、2021年から2026年までの約5年分を集計しました。使ったコードとデータの取得方法も後半で示すので、読んだ後に自分で最新まで確かめられます。オプションを触ったことがない人でも追えるよう、用語から順番に説明します。

## オプションとは何か。コール・プット・プレミアム

オプションは「将来、決めた価格で売り買いする権利」を売買する仕組みです。基本は2種類だけです。

- コール: 将来、決めた価格で買う権利
- プット: 将来、決めた価格で売る権利

このとき「決めた価格」を権利行使価格、「いつまで」を満期と呼びます。そして権利を手に入れるために最初に払う代金がプレミアムです。オプションを買った側は、権利を使っても使わなくてもよく、損失は最初に払ったプレミアムまでに限られます。権利であって義務ではない、というのがオプションの一番の特徴です。

損益の形を見ると、性格がよく分かります。

![コール・プット・現物の損益図](/images/btc-iv-premium-vs-realized-vol/fig1_payoff.png)

左が現物、つまりBTCをそのまま持っている場合です。価格に比例して損益が動きます。真ん中がコール買いで、価格が権利行使価格を超えたぶんだけ利益が出て、外れてもプレミアム分の損で止まります。右がプット買いで、価格が下がるほど利益が出ます。プットは、下落したときに利益が出るので、現物を持っている人にとっては下落保険に近い働きをします。

ここで一つ注意です。プットは下落保険に使えますが、オプションそのものは投機にもヘッジにも使えます。プットを買う人が全員「下がると予想している投機家」とは限らず、現物を守りたい人の保険であることも多い、という点は後の記事で効いてきます。

## なぜBTCオプションは「高い」のか

オプションのプレミアムは、いくつかの要素で決まります。原資産の価格、権利行使価格、満期までの時間、金利、そして予想変動率です。このうち、同じBTC・同じ満期・同じ行使価格でプレミアムの大小を決める主役が、予想変動率です。

予想変動率は、これから原資産がどれくらい激しく動きそうか、という市場の見積もりです。大きく動きそうなら、保険が使われる可能性も上がるので、プレミアムは高くなります。ビットコインは株や為替に比べて値動きが激しいため、予想変動率が高く、プレミアムも高くなりやすいのです。

つまり「BTCオプションは高い」の正体は、多くの場合この予想変動率の高さです。ただ、それが「払いすぎ」だったかどうかは、金額の絶対値を見ても分かりません。予想変動率が高くても、実際にそれ以上動いたなら妥当な値付けです。判断するには、市場の予想と、その後に本当に起きた変動を並べて比べる必要があります。

## IV・DVOL・実現ボラを整理する

比べるために、3つの言葉を区別します。

- IV（予想変動率）: オプションの価格から逆算した、市場が織り込む将来の変動の大きさ。個別のオプション1つずつに対して計算されます
- DVOL: Deribitが複数のBTCオプションから算出する、今後30日間の予想変動率を表す指数。個別のIVそのものではなく、市場全体の予想を1本の数字に集約したものです
- 実現ボラティリティ: 過去の価格から計算した、実際に起きた変動の大きさ

今回の検証で「市場の予想」として使うのがDVOLです。DVOLは30日先を見た指数なので、これと「その後30日で実際に起きた変動（実現ボラ）」を突き合わせれば、予想が高すぎたかどうかを事後的に採点できます。DVOLも実現ボラも、年率換算したパーセントで表します。

一点、先に断っておきます。DVOLは値動きの「大きさ」を測る指数で、下落だけでなく上昇の大きな動きも織り込みます。つまり厳密には下落専用の保険料ではありません。この記事で「保険料」と呼ぶのは、値動き全体に付いた値段のたとえだと思ってください。下落方向に絞った価格の偏り（スキュー）は、続編で扱います。

DVOLがどう動いてきたかを、BTC価格と重ねて見ておきます。

![BTC価格とDVOL](/images/btc-iv-premium-vs-realized-vol/fig2_price_dvol.png)

価格が急落した局面でDVOLが跳ね上がっているのが分かります。市場が「これから荒れそうだ」と身構えるときに、予想変動率が上がる、という関係です。ではその予想は、当たっていたのでしょうか。

## 検証：予想変動率は、その後の実現より高かったのか

やることはシンプルです。ある日のDVOLと、その日から30日間に実際に起きた実現ボラを、日ごとに並べて引き算します。

- データ: DeribitのDVOL日足と、BTC-PERPETUAL（満期のない先物）の日足価格（2021年3月から2026年7月）
- 実現ボラ: 日次の対数リターン（価格の比の対数をとった日次の変化率）の標準偏差を年率換算します。ばらつきは時間の平方根で伸びるので、日次の標準偏差に日数の平方根を掛けると年率になります。暗号資産は24時間365日動くので365で換算します
- 各日について、その後30暦日の実現ボラを計算し、DVOLから引く

計算の中心は、この「各日から前を向いて30日ぶんの実現ボラを取る」ところです。

```python
import numpy as np
import pandas as pd

df["logret"] = np.log(df["close"]).diff()  # 日次の対数リターン

# 各日 t について「その翌日から30暦日先まで」の実現ボラ（年率%）を前を向いて計算
def forward_rv(t):
    seg = df.loc[(df["date"] > t) & (df["date"] <= t + pd.Timedelta(days=30)), "logret"]
    return seg.std() * np.sqrt(365) * 100 if len(seg) >= 20 else np.nan

df["fwd_rv"] = df["date"].map(forward_rv)
df["premium"] = df["dvol"] - df["fwd_rv"]  # 予想 − 実現。プラスなら予想が実現を上回った
```

ここで `premium`（予想と実現の差）は、便宜的な呼び名です。これはDVOLと実現ボラという2つの変動率の差であって、オプションの金額としてのプレミアムそのものでも、売買して得られる損益でもありません。あくまで「市場の予想は、その後の実現より高かったか」を測るための差だと考えてください。

まず全期間をまとめて見ます。市場の予想（DVOL）と、その後に起きた実現ボラの推移です。

![DVOLと将来30日の実現ボラ](/images/btc-iv-premium-vs-realized-vol/fig3_dvol_vs_rv.png)

青い予想の線が、オレンジの実現の線より上にあることが多い、という傾向が見えます。差を取ってヒストグラムにすると、もっとはっきりします。

![予想と実現の差の分布](/images/btc-iv-premium-vs-realized-vol/fig4_premium_hist.png)

分布の山は0より右、つまり「予想のほうが高かった」側に寄っています。数字にすると、検証した1926日のうち、**DVOLが実現ボラを上回った日は73.7%**でした。差の平均はプラス9.3ポイント、中央値はプラス10.8ポイントです。平均のDVOLが60.9%、平均の実現ボラが51.5%なので、市場はその後の実現より年率で約9ポイント（比にするとおよそ2割）高めに変動を見積もっていたことになります。

予想と実現をそのまま散布図にすると、両者の関係と、予想が上振れしがちな様子が見えます。

![予想と実現の散布図と回帰線](/images/btc-iv-premium-vs-realized-vol/fig5_scatter.png)

点は右肩上がりに散らばっていて、相関は0.64でした。予想が高い時期は実現も高い、という関係はあります。ただし灰色の点線（予想＝実現の線）より下に点が多く、回帰の傾きは0.62でした。傾きが1より小さいのは、DVOLが高いときほど、実際の変動を上回って予想しがちだったことを表します。ここまでをまとめると、ビットコインの保険料は、この5年では平均的に少し割高だった、と言えます。

## 平時は割高、暴落時は逆転する

ここで話を終えると、危ない結論に飛びつきそうになります。「平均的に割高なら、保険を売る側（プットの売り手など）になれば儲かるのでは」という発想です。これが正しいかを確かめるために、局面を分けて見ます。

まず、DVOLの水準で3つの局面に分けて、平均のプレミアムを比べます。

![局面別の平均プレミアム](/images/btc-iv-premium-vs-realized-vol/fig6_tercile.png)

DVOLが低い局面ではプラス3.0ポイント、中くらいでプラス8.3ポイント、高い局面ではプラス16.8ポイントでした。予想変動率が高く跳ねている時期ほど、平均で見ると予想が実現を上回っていた、という結果です。これだけ見ると「変動が大きいと市場が身構えている時に保険を売れば、より儲かる」と読めてしまいます。

ところが、本当に大きく下げた局面だけを取り出すと、絵が反転します。ここでは、各日から前を向いた30暦日のあいだに、最安値まで何%下げたかを見て、下落が大きかった下位10%（この検証ではおよそ22%以上下げた場面）を「暴落局面」、残りを「平時」と呼び分けています。暴落局面に絞ると、**平均のプレミアムはマイナス4.9ポイント**でした。マイナスは、予想（DVOL）を実現が上回ったこと、つまり保険が実際に効いたことを意味します。平時のプラス10.9ポイントと比べると、まさに逆転しています。なお30日の窓を毎日ずらして数えているので、1回の暴落が何日ぶんも重複して数えられている点は割り引いて見てください。検証期間で最も大きかった30日の下落は、43.6%でした。

急変した1つの局面を拡大すると、この逆転の瞬間が見えます。

![急変ケース（2021年5月前後）](/images/btc-iv-premium-vs-realized-vol/fig7_case.png)

2021年5月、ビットコインが短期間で大きく崩れた場面です。下落が始まると、実現ボラ（オレンジ）がDVOL（青）を追い抜いています。ふだんは払いすぎに見えた保険料が、この数十日については安すぎた、ということです。

ここから読み取れるのは、次の非対称です。ビットコインの保険料は、平時には平均的に割高でした。しかし、暴落という滅多に来ない場面では、予想を超える変動が起きて、保険が価値を持ちました。だから「平均的に割高であること」と「保険を売れば安全に儲かること」は、まったく別の話です。

念のため補いますが、ここで見ているのはあくまで予想変動率と実現ボラの差であって、実際に保険を売ったときの損益ではありません。それでも、平均がプラスに寄る一方で暴落時に大きくマイナスへ振れる、というこの差の形は、変動リスクを引き受ける側の損益がたどりやすい形と重なります。平均のプラスは、滅多に来ない大きなマイナスを、平時の小さなプラスがならしているように見える、ということです。実際に売買した場合の損益は、権利行使価格・スキュー・手数料・建て替え方といった、ここで測っていない要素で大きく変わります。

## わかったことと、この検証の限界

この5年のビットコインで分かったことを整理します。

- 市場が見積もる変動率（DVOL）は、その後の実現ボラより高いことが多かった（73.7%の日、平均でプラス9.3ポイント）
- 予想変動率が高い局面ほど、平均で見ると予想が実現を上回っていた
- ただし本当に大きく下げた局面では逆転し、保険が価値を持った（平均マイナス4.9ポイント）
- 「平均的に割高」と「保険を売れば安全」は別。変動リスクを引き受ける側は、平時の小さな利益と引き換えに、暴落時の大きな損失にさらされる

この非対称は、オプションを「相場を当てる道具」ではなく「将来の不確実性に値段をつける市場」として見ると腑に落ちます。平時に保険料を払う人はコストを負担しがちでも、急落の損失を避けたい人にとっては、その値段には合理性があります。

最後に、この検証の限界も正直に書いておきます。

- 実現ボラは「値動きの大きさ」であって、オプションの実際の損益とは同じではありません。手数料、売買のスプレッド、満期ごとの建て替え（ロール）、権利行使価格による予想変動率の偏り（スキュー）、流動性、取引所そのもののリスクは、今回の計算に含めていません
- 30日の窓を毎日ずらして重ねて数えているため、隣り合う日のデータは互いに似ています（自己相関）。そのため「統計的にどれくらい確からしいか」を強く主張することはしません
- これはあくまで、ある取引所の、ある期間の、ある計算方法での結果です。別の期間や別の定義では数字は変わります

それでも、「保険料は高すぎるのか」という問いに対して、平時は割高・急変時は合理的という非線形の答えが、この期間のこの測り方で見えてきたのは面白いところでした。統計的にどれだけ確かな性質かは、窓の重なりを避けた検証などで別途詰める必要があります。

データはDeribitの公開API（DVOLとBTC-PERPETUAL価格）から取得し、集計とグラフの生成コードも手元に残しています。次の章では、このオプション市場から「参加者の恐怖」を読み取れるのか、その恐怖から暴落を予知できるのかを試します。

（データ出典: Deribit公開API。本記事では生データを再配布せず、集計値とグラフのみを掲載しています。取得日時は2026年7月22日UTCです。数値はすべて筆者の集計で、誤りを含む可能性があります。）
